dhcp(动态主机配置协议)是一个局域网的网络协议.使用UDP协议进行工作。
	主要用途.
		(1)给内部网络或网路服务供应商自动分配IP地址
		(2)给用户或者内部网络管理员作为对所有计算作中央管理的手段。

dhcp有三个端口：
	UDP 67 和 UDP 68为正常的DHCP服务端口，分别作为DHCP Server 和DHCP Client 的服务端口
	564号端口用于DHCPv6 Client,而不用于DHCPv4，是为DHCP failover（失效备援）服务，用来做"双机热备"的。
	
dhcp服务作用：
		为大量客户端自动分配地址，提供几种管理
		减轻管理和维护成本,提高网络配置效率

dhcp原理:
	1.客户端寻 XXX 器进行IP请求
	2. 服务端提供地址信息
	3. 客户端接受并广播(客户端选择IP)
	4. 服务端确认租约(DHCPACK)
		如果DHCP客户机无法找到DHCP服务器，将从TCP/IP的B类网络169.254.0.0/16中挑选一个IP作为自动的IP，每隔5分钟继续尝试与DHCP服务器通信。
	5. 客户端重新登录（IP地址）
	6. 服务端确认
	
DHCP客户端续租:
	DHCP客户机在租期过去50%的时候，直接向为其提供IP地址的DHCP服务器发送DHCPREQUEST消息包，如果客户机接收到该服务器回应的DHCPPACK消息包，客户机就根据包中所提供的新的租期 和 其他已经更新的TCP/IP参数更新自己的配置。
	如果没有接收到该服务器的回复，客户机会继续使用现有ip。
	 如果在租期过去50%的时候没有更新，则DHCP客户机在租期过去87.5%的时候再次向为其提供IP地址的DHCP服务器联系
	 如果还不成功，到租约的100%的时候，DHCP客户机必须放弃这个IP地址，重新申请。
	 如果此时无DHCP服务器可用，DHCP客户机会使用169.254.0.0/16中随机的一个地址，并且每隔5分钟再进行尝试。
