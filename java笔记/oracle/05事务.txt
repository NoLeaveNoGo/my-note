数据库事务的目的:
    (1)多个程序并发访问时，提供一个隔离的方法，放置彼此操作相互干扰。
    (2)提供一个从失败中恢复正常的方法，提供了数据库即使在异常状态下也能保持数据一致性
   
特性：ACID 原子性、一致性、隔离性、持久性

事务的四种隔离级别：
    read uncommit：读未提交:一个事务可以读取到另一个事务的未提交数据(存在脏读问题)
        
    read commit: 读已提交:要等另一个事务提交后才能读取数据
       但若另一个事务对数据有更新,那么在它提交前后，读取到的数据不一致。(不可重复读问题)
    
    可重复读: 在开始读取数据(事务开启)时，不再允许修改操作。
        只是针对UPDATE操作，但另一个事务有可能insert操作(幻读)
        幻读: 事务开始后，另一个事务有insert操作，并提交。
        
    序列化：在该级别下，事务串行化顺序执行
