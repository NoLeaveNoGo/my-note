1.多线程调试:
	* 在IDE里断点停下来的时候，可以切换到另外的线程中，跑其他的代码，不会互相影响。当然，这里是有个开关的，在Eclipse里默认开启，
	* 但是在IntelliJ IDEA里默认是没有开启的。
	* 简单设置可以直接在断点处，将suspend改成Thread
	* 在一个线程断点停下来之后，可以在Frames窗口切换到其他线程进行执行。
	
2. Drop Frame 
	* 关闭当前调用栈，然后重新调用一遍 。
	* 近似于后退执行，但被方法修改的外部引用值无法恢复到进入该方法栈时的那种状态。
	
3. 条件断点
	* 在断点处右击，可以设置Condition，只有Condition满足断点才会停下来
