java对象序列化
1.一般情况下，只有jvm处于运行时，对象才可能存在，即对象的生命周期不会比JVM的生命周期更长。但是在现实应用中，就可能药酒JVM停止运行后能够保存(持久化)指定的对象，并在将来重新读取被保存的对象。java对象序列化就能够帮助我们实现这些功能。

2.实现Serializable接口，通过对象流(ObjectOutoutStream、ObjectInputStream来持久化对象)
Serializable的作用：如果被写对象的类型是String、数组[obj.isArray()]、Enum、Serializable,那么就可以对该对象进行序列化。否则会抛出NotSerializableException。

3.默认序列化机制：
	不仅会序列化当前对象本身，还会对该对象引用的其它对象也进行序列化，同样地，这些其它对象引用的另外对象也将被序列化，以此类推。所以，如果一个对象包含的成员变量是容器类对象，而这些容器所含有的元素也是容器类对象，那么这个序列化的过程就会较复杂，开销也较大。

	1.transient关键字：当某个字段被声明为transient后，默认序列化机制就会忽略该字段。
