简单工厂模式
	 


1.类图
1.1表示类的框分成以下几层：
	类名
	属性清单(+代表public、-代表private、#代表protected)
	方法清单
	性质形单
1.2如果一个类有内部成员类，它的类图就会有五层。

2.类图中的关系
	在类与类之间，会有连线指明它们之间的关系。
	主要有以下几种关系：
		一般化关系：类、接口的继承关系、类与接口的实现关系。连线由子类指向父类，与继承方向相关
		关联关系：一个类知道另一个类的属性和方法。可以是双向的
	

	
	
	当调用方法时，传给方法的参数就是实际参数（实参）
	
	在定义方法时，方法名后括号内的参数就是形式参数。
